To implement full functionality for access token issuance and refresh token handling using `oxide-auth` with Axum in Rust, we need to fully implement the token endpoint and the refresh token flow. This example will cover:

1. **Login & Authentication** – Simulated simple login flow.
2. **OAuth2 Token Issuance** – Issuing an access token and a refresh token.
3. **Refresh Token Handling** – Allowing a client to use a refresh token to get a new access token.

Here’s a full working example with refresh token functionality.

### Cargo.toml

Ensure you have the following dependencies in your `Cargo.toml` file:

```toml
[dependencies]
axum = "0.6"
tokio = { version = "1", features = ["full"] }
oxide-auth = { version = "0.5", features = ["refresh"] }
serde = { version = "1", features = ["derive"] }
serde_json = "1.0"
uuid = { version = "1", features = ["v4"] }
```

### Full Example Code

```rust
use axum::{
    routing::{get, post},
    extract::{Form, Query, State},
    response::IntoResponse,
    http::StatusCode,
    Router,
};
use oxide_auth::frontends::simple::endpoint::{Generic, GenericEndpoint};
use oxide_auth::primitives::prelude::*;
use oxide_auth::code_grant::frontend::*;
use oxide_auth::primitives::authorizer::AuthMap;
use oxide_auth::primitives::issuer::TokenMap;
use oxide_auth::primitives::registrar::{BoundClient, ClientMap};
use oxide_auth::endpoint::OAuthError;
use serde::{Deserialize, Serialize};
use std::sync::Arc;
use tokio::sync::Mutex;

#[tokio::main]
async fn main() {
    let mut registrar = ClientMap::new();

    // Register a client with refresh token capabilities
    registrar.register_client(BoundClient {
        client_id: "client".to_string(),
        redirect_uri: "http://localhost:3000/callback".parse().unwrap(),
        client_secret: Some("secret".to_string()),
        default_scope: Some("default".parse().unwrap()),
        additional_scopes: vec![],
        allowed_grant_types: vec![GrantType::AuthorizationCode, GrantType::Refresh],
        token_type: BearerToken::RefreshToken, // Enables refresh token issuance
    });

    let oauth_endpoint = Arc::new(Mutex::new(Generic {
        registrar,
        authorizer: AuthMap::new(),
        issuer: TokenMap::new(),
        solicitor: AllowSolicitor,
    }));

    let app = Router::new()
        .route("/", get(home))
        .route("/login", post(login))
        .route("/auth", get(auth))
        .route("/callback", get(callback))
        .route("/token", post(token))
        .route("/refresh", post(refresh_token))
        .with_state(oauth_endpoint);

    let addr = "127.0.0.1:3000".parse().unwrap();
    println!("Listening on http://{}", addr);
    axum::Server::bind(&addr)
        .serve(app.into_make_service())
        .await
        .unwrap();
}

async fn home() -> &'static str {
    "Welcome to the OAuth2 Login Example with Refresh Token"
}

#[derive(Deserialize)]
struct LoginForm {
    username: String,
    password: String,
}

// Mock login function that redirects to /auth on success
async fn login(Form(data): Form<LoginForm>) -> impl IntoResponse {
    if data.username == "user" && data.password == "password" {
        // In a real app, you'd authenticate the user here.
        Redirect::temporary("/auth")
    } else {
        Redirect::temporary("/")
    }
}

// Mock authorization handler (e.g., after login success)
async fn auth() -> &'static str {
    "Authorization endpoint"
}

// Mock OAuth2 callback handler (after authorization code grant)
async fn callback() -> &'static str {
    "Callback endpoint (after auth code grant)"
}

#[derive(Deserialize)]
struct TokenRequest {
    grant_type: String,
    code: Option<String>,     // For authorization code grant
    refresh_token: Option<String>, // For refresh token grant
    client_id: String,
    client_secret: String,
}

async fn token(
    State(oauth_endpoint): State<Arc<Mutex<GenericEndpoint>>>,
    Form(request): Form<TokenRequest>,
) -> impl IntoResponse {
    let mut endpoint = oauth_endpoint.lock().await;

    // Determine if this is an authorization code or refresh token request
    match request.grant_type.as_str() {
        "authorization_code" => {
            if let Some(code) = request.code {
                match endpoint.access_token_from_code(&code, &request.client_id, Some(&request.client_secret)) {
                    Ok(token_response) => {
                        // Return the access token and refresh token
                        (
                            StatusCode::OK,
                            serde_json::to_string(&token_response).unwrap(),
                        )
                    }
                    Err(OAuthError::InvalidGrant) => {
                        (StatusCode::BAD_REQUEST, "Invalid authorization code".into())
                    }
                    _ => (StatusCode::INTERNAL_SERVER_ERROR, "Failed to issue token".into()),
                }
            } else {
                (StatusCode::BAD_REQUEST, "Authorization code missing".into())
            }
        }
        "refresh_token" => {
            if let Some(refresh_token) = request.refresh_token {
                match endpoint.refresh_token(&refresh_token, &request.client_id, Some(&request.client_secret)) {
                    Ok(token_response) => {
                        // Return the new access token
                        (
                            StatusCode::OK,
                            serde_json::to_string(&token_response).unwrap(),
                        )
                    }
                    Err(OAuthError::InvalidGrant) => {
                        (StatusCode::BAD_REQUEST, "Invalid refresh token".into())
                    }
                    _ => (StatusCode::INTERNAL_SERVER_ERROR, "Failed to issue token".into()),
                }
            } else {
                (StatusCode::BAD_REQUEST, "Refresh token missing".into())
            }
        }
        _ => (StatusCode::BAD_REQUEST, "Invalid grant type".into()),
    }
}

async fn refresh_token(
    State(oauth_endpoint): State<Arc<Mutex<GenericEndpoint>>>,
    Query(params): Query<TokenRequest>,
) -> impl IntoResponse {
    let mut endpoint = oauth_endpoint.lock().await;

    if let Some(refresh_token) = params.refresh_token {
        match endpoint.refresh_token(&refresh_token, &params.client_id, Some(&params.client_secret)) {
            Ok(token_response) => {
                // Return the new access token after refresh
                (
                    StatusCode::OK,
                    serde_json::to_string(&token_response).unwrap(),
                )
            }
            Err(OAuthError::InvalidToken) => {
                (StatusCode::BAD_REQUEST, "Invalid refresh token".to_string())
            }
            _ => {
                (StatusCode::INTERNAL_SERVER_ERROR, "Something went wrong".to_string())
            }
        }
    } else {
        (StatusCode::BAD_REQUEST, "Refresh token missing".to_string())
    }
}
```

### Explanation of the Changes

1. **Client Registration:**
   - The client is registered with support for both `AuthorizationCode` and `Refresh` grant types. This allows the client to first obtain an access token via the authorization code and later use a refresh token to request new access tokens without re-authenticating.

2. **Login and Authorization:**
   - The `/login` and `/auth` endpoints simulate a simple login and authorization flow. You can expand these to handle actual user sessions and authorization logic.

3. **Token Request Handling:**
   - The `/token` endpoint handles requests for both `authorization_code` and `refresh_token` grant types.
     - For `authorization_code`, it validates the code and client credentials and returns an access token and a refresh token.
     - For `refresh_token`, it validates the refresh token and issues a new access token.

4. **Refresh Token Handling:**
   - The `/refresh` endpoint allows clients to refresh their access token using a valid refresh token. The same logic is integrated into the `/token` endpoint for `grant_type=refresh_token`.

5. **Token Request Format:**
   - Both access token requests and refresh token requests expect a `POST` form with parameters:
     - `grant_type`: Either `authorization_code` or `refresh_token`.
     - `client_id`: The registered client ID.
     - `client_secret`: The client secret.
     - For authorization code grant: `code` (the authorization code).
     - For refresh token grant: `refresh_token` (the refresh token).

### Token Response

When a valid token request is made, the response will contain:

```json
{
  "access_token": "new_access_token_value",
  "token_type": "bearer",
  "expires_in": 3600,
  "refresh_token": "new_refresh_token_value"
}
```

### Running the Example

1. Run the server using `cargo run`.
2. Simulate the login flow by posting valid credentials to `/login`.
3. Test token issuance by posting to `/token` with valid authorization code and client credentials.
4. Test the refresh flow by posting to `/refresh` with a valid refresh token.

This covers full functionality for both obtaining access tokens and refreshing them.
